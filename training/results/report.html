Executive Summary

What was run

Two experiments were completed from the saved repository outputs: a similarity-based ranker and a preference-based ranker. Both used the same grouped task setup, GRU text encoder, hinge loss, and candidate text fields from the saved configs.

2 experiments Similarity vs Preference Grouped ranking by task_id Reduced 5-epoch CPU runs

Main outcome

The preference model was the strongest run on both best validation Score@1 (0.3125) and test Score@1 (0.2615). The similarity model was slightly stronger on test MRR (0.3898 vs 0.3774).

These were reduced runs rather than full-scale training, so the report is best interpreted as a solid implementation-level comparison rather than a final benchmark claim.

Observations and Caveats

What stands out

  • The preference model delivered the strongest validation Score@1 (0.3125) and the strongest test Score@1 (0.2615).
  • The similarity model retained a small edge on test MRR (0.3898).
  • The absolute gap in test Score@1 between the two runs was small (0.0051), so the ranking order is directionally useful but not decisive.

How to interpret the results

  • These results come from reduced runs, not exhaustive tuning or multiple seeds.
  • The report reflects saved experiment outputs only; it does not infer unrun settings or hidden gains.
  • Validation curves suggest some instability after the best epoch, so longer runs would need careful checkpoint selection and probably broader tuning.
